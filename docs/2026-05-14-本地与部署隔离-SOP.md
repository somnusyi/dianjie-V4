# 本地开发与部署隔离 SOP

> 2026-05-14 立。解决问题: 部署生产时跑 `pnpm install` / `prisma generate` / `pnpm web build`
> 会写主仓库的 `node_modules` 和 `.next`, 导致本地 dev 进程被打扰 (tsx watch 崩 / next dev 找不到 chunks)。
>
> 一天内发生过 3 次, 每次都得清缓存 + 重启 dev, 浪费时间。

## 仓库布局

```
~/Desktop/dianjie-V4/
├── dianjie-V4/           ← 主仓库 (本地开发 + 写代码 + git commit)
│   ├── .git/             共享 git 历史
│   ├── apps/
│   ├── node_modules/     主仓库独立 (本地 dev 用)
│   └── .next/            主仓库独立 (next dev 用)
│
└── dianjie-V4-deploy/    ← 部署 worktree (detached HEAD, 只跑 build)
    ├── .git              指向主仓库 .git (file 引用)
    ├── apps/
    ├── node_modules/     worktree 独立 (build 用)
    └── .next/            worktree 独立 (production build 用)
```

两个目录共享 git 历史和 tracked 文件,但 `node_modules` / `.next` / `apps/api/dist` 物理隔离。

## 日常工作流

### 1. 本地开发 (主仓库)

```bash
cd ~/Desktop/dianjie-V4/dianjie-V4
pnpm --filter @dianjie/api --filter @dianjie/web -r --parallel dev
# api → :4444, web → :3200
# 改代码 HMR 自动生效
```

CMB Flask 微服务单独起 (port 5001):
```bash
cd apps/cmb && python3 app.py  # 或者你之前的方式
```

### 2. 改完一个功能 → 提交

```bash
cd ~/Desktop/dianjie-V4/dianjie-V4
git add ...
git commit -m "..."
git push origin main      # 推到远程
```

### 3. 部署生产 (worktree)

**主仓库 dev 进程保持运行,完全不动**。所有 build 操作在 worktree 跑:

```bash
cd ~/Desktop/dianjie-V4/dianjie-V4-deploy

# 同步 main 最新 (主仓库 push 后)
git fetch origin
git checkout origin/main

# 装 deps (lockfile 没变就秒过, pnpm hardlink 不重新下)
pnpm install --frozen-lockfile

# build (写入 worktree 自己的 .next 和 dist, 不碰主仓库)
pnpm --filter @dianjie/db exec prisma generate
pnpm --filter @dianjie/api exec tsc            # tsc emit 即使有 pre-existing error
pnpm --filter @dianjie/web build

# rsync + reload (照常)
export V4_SSH_PASSWORD='...'
sshpass -p "$V4_SSH_PASSWORD" rsync -avz --delete \
  -e 'ssh -o StrictHostKeyChecking=no' \
  apps/api/dist/ \
  root@116.62.32.162:/app/dianjie-v4/apps/api/dist/

sshpass -p "$V4_SSH_PASSWORD" rsync -avz --delete \
  -e 'ssh -o StrictHostKeyChecking=no' \
  apps/web/.next/standalone/apps/web/ \
  root@116.62.32.162:/app/dianjie-v4/apps/web/apps/web/

sshpass -p "$V4_SSH_PASSWORD" rsync -avz --delete \
  -e 'ssh -o StrictHostKeyChecking=no' \
  apps/web/.next/static/ \
  root@116.62.32.162:/app/dianjie-v4/apps/web/apps/web/.next/static/

sshpass -p "$V4_SSH_PASSWORD" ssh root@116.62.32.162 \
  "pm2 reload dianjie-v4-api dianjie-v4-web --update-env"
```

**部署期间主仓库的 `localhost:3200` 一直可用,继续开发,完全无感**。

## 关键规则

1. **主仓库永远不跑 `pnpm install` / `prisma generate` / `pnpm web build`** —— 这些命令在 worktree 跑
2. **worktree 不跑 dev** —— 它只是部署的临时构建环境
3. 两个仓库的 `node_modules` 完全独立,主仓库装新依赖后,worktree 下次部署前要 `pnpm install` 同步
4. 主仓库改完代码 → `git commit + git push` 后,worktree 才能拿到最新代码 (走 `git fetch + checkout origin/main`)
5. **不能在 worktree 里 commit** (detached HEAD,commit 不归属任何分支)

## 一次性 setup (已完成)

